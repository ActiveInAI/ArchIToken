二、已实现的平台级能力

前端工作台

  • /app/modules 与 /app/modules/[moduleId] 动态模块入口已实现。
  • 14 模块共享 ModuleWorkbenchShell，包含左侧 rail、模块目录、主工作区、审计抽屉和主题切换。
  • module-registry.ts 覆盖模块、子域、交付物、workflow、agent gates、任务、审批、风险、文件类型、标准和数据对象。
  • module-operations.ts 为 14 个模块提供业务功能卡片和操作按钮。
  • 文件系统、预览、Office/CAD/BIM/Archive viewer、生命周期事务、审批、审计、OpenClaw dock 均已有前端或 adapter 边界。

后端与运行时

  • Rust gateway 已提供 auth、projects、modules、files、transactions、audit、generation、artifacts、openBIM、assets、conversion jobs、viewer commands、skills、MCP tools、knowledge sources 等 API。
  • Rust shared registry 覆盖 14 模块，并有 registry 顺序、命名和生产制造存在性测试。
  • Python agent orchestrator 为 14 模块生成 Planner / Generator / Evaluator 图。
  • workers 覆盖 CAD、IFC/openBIM、GIS、document、image、AI generation、source build、steel platform、engine server 等 adapter 合同。
  • 后端遵循 Router、CDE、审批、审计、ObjectStore、RuntimeContext、RBAC 和 fail-closed adapter 边界。

六、最快上线时间判断

最快可上线窗口

  • 内部演示/开发预览：最快 2026-05-27 晚间 AWST。前提是修复前端 lint 2 个错误、补齐 worker Pillow 测试依赖或调整测试 extra、跑通 bun run build。
  • 私有 MVP 试点：建议 2026-06-03 至 2026-06-05。范围限定为平台底座、计划管理、施工管理、数字孪生只读、文件/审批/审计。
  • 外部商业生产：不建议早于 2026-06-23。需要真实部署、备份、监控、安全、外部 adapter、专业签审和法律/许可证证据。

上线口径

  • 可以说“已具备 14 模块统一工作台和部分业务模块私有试点能力”。
  • 不能说“全模块生产完成”“全专业合规”“可施工/可报审/可验收”。
  • 数字孪生、造价、深化、生产制造应标注为“专业复核 required”或“试点”。
  • 任何支付、电子签章、CAD/BIM 私有格式转换、AI 生成媒体都必须有真实 adapter 证据后再开放给客户。
