方案 A 的边界

  • 可以支撑内部研发、业务流程梳理、BOM 数据底座、Agent 编排验证、100 用户试点和移动端现场采集试点。

7. 128-160 Agent 容量模型

9. 到货验收与上线检查

硬件验收

  • 核对 CPU、内存、硬盘、NVMe、主板、BMC、网卡、电源、序列号和保修。
  • BIOS / BMC 固件升级并导出配置备份。
  • ECC、内存通道、NVMe SMART、HDD SMART、风扇和电源冗余检查。
  • 72 小时 burn-in：CPU、内存、fio、ZFS scrub、iperf3 10GbE、断电恢复。

系统验收

  • Kubernetes 节点、容器镜像仓库、Ingress、证书、备份任务和恢复演练。
  • PostgreSQL、对象存储、向量库、BOM 主数据、Ontology 字段唯一性。
  • JumpServer MFA、SSH/RDP/HTTPS 接入、命令审计、录像、日志外送。
