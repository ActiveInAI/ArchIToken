## 5 · Zed + Claude Code 集成 · 新增独立文档

AIA 原话:"**我想在 zed 中使用 claude code·应该怎么打开使用?现在切窗口太麻烦**"

根治方案文档: `ZED-CLAUDE-CODE-SETUP.md`

核心配置(15 分钟搞定):

```bash
# 1. 确认 claude CLI
claude --version

# 2. 编辑 ~/.config/zed/settings.json
# 加入:
{
  "agent": {
    "external_agents": {
      "claude_code": {
        "command": "claude",
        "args": []
      }
    }
  }
}

# 3. 在 Zed 里 Ctrl+? 打开 Agent Panel
# 4. 切到 claude_code 外部 agent
# 5. 直接在 Zed 窗口里对话·不用切浏览器
```

详细完整文档在 `ZED-CLAUDE-CODE-SETUP.md` · 7 大节 · 包含常见问题和 Insome VPN 代理配置。

## 6 · 决策建议: 本地顶端 LLM 选哪个

现在有三个候选:

| 候选 | 大小 | 格式 | 优势 | 劣势 |
|---|---|---|---|---|
| nvidia/Gemma-4-31B-IT-NVFP4 | 1.2G | NVFP4 | GB10 原生全速·小 | 能力弱于 32B |
| IndustrialCoder-Thinking-32B-FP8 | 估 30G+ | FP8 | ⭐ 32B·Thinking·代码强 | 大·需更多 VRAM |
| 两者并存·按任务路由 | 31G+ | 混合 | 最佳路由 | LLM Router 复杂度+1 |

**推荐: 两者并存 + 任务路由**

```
# apps/api/src/llm_router.rs 的路由决策:
任务类型 == "code_generation" || "agent_planning" || "architecture"
  → IndustrialCoder-Thinking-32B-FP8
任务类型 == "quick_completion" || "simple_query" || "UI_reply"
  → Gemma-4-31B-IT-NVFP4
任务类型 == "edge_device"
  → Ternary-Bonsai-8B (1-bit·CPU 可跑)
任务类型 == "顶端质量"
  → runtime provider alias
```
