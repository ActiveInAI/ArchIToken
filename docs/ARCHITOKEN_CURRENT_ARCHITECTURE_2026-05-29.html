1. 总体定位

ArchIToken = AEC AI-Native + Harness Engineering + OpenBIM CDE Workflow OS

平台不是 Revit、Tekla、PKPM、广联达、中望、Siemens Building X 等单点产品的克隆或直接替代，而是位于这些工具之上的开放工程智能层、CDE 文件运行时、AI 门禁系统和可私有化部署的工作流 OS。

工程真源

GitHub 仓库文档、Schema、代码、CI 和证据文件是唯一真源。聊天上下文和临时讨论只能作为输入。

开放语义

CAD/BIM/CIM/GIS 数据底座以 buildingSMART openBIM 为核心: IFC、IDM、bSDD、BCF、IDS、Validate 与 OpenCDE API。

可审计交付

文件、模型、AI 输出、工具调用、审批、版本和归档都必须产生可追溯证据链。

2. 非协商架构原则

  • Registry 替代 Enum: 模块、Agent、Tool、Model、Router、Schema、Geometry Kernel、Renderer、Workflow、Rule 都走注册机制。
  • AI 调用必须经过内部 ModelRouter / InferenceRouter / ToolRouter / WorkflowRouter，业务逻辑禁止直连外部模型 API。
  • Generator 与 Evaluator 必须分离，交付链固定为 Planner -> Generator -> Evaluator -> RuleChecker -> SchemaValidator -> Approver。
  • 开放格式走原生/open runtime；私有或复杂格式走 worker、sidecar、授权适配器或企业服务，不用前端派生文件冒充真实支持。
  • 缺少专业或监管来源时，只能输出经验建议，不能输出合规、可施工、可报审、可验收、可清关、可入账、可支付、可发布等结论。
  • GPU-first: WebGPU 是浏览器交互式渲染/计算默认路线，WebGL/CPU 只能是有失败证据的 fallback。
  • NVIDIA 目标硬件必须采用 NGC CUDA/CUDA-DL、Container Toolkit、GPU Operator、DCGM、CUDA、OptiX、TensorRT/Triton 等认证或支持栈。
  • GPL/AGPL/LGPL/SSPL/BUSL/Commons Clause 不进入核心分发边界；强能力项目可通过外部服务或隔离 sidecar 使用。

3. 八层全栈架构

L7
Consumer Surfaces: Next.js 16.2.6 + React 19.2.5 PWA；React Native 0.85.1 移动端；Tauri 2.10.1 桌面实验路线。
L6
Auto-Generated SDK: OpenAPI 3.1 生成 TypeScript、Python、Rust、Go、Java、Swift、Kotlin 等 SDK。
L5
Unified API Gateway: Rust axum 0.8.9 REST/SSE + tonic 0.14.5 gRPC + MCP Server；负责鉴权、限流、审计、API 边界。
L4
Agent Orchestration: LangGraph 1.1.8 为 Python 主编排；VoltAgent / PanAI 等只作为 Router 边界后的候选接入。
L3
Harness Core: Rust 1.95.0 + tokio + sea-orm 2.0.0-rc.38 + utoipa 5.4.0；承载 Router、Schema gate、状态保护、文件和高并发 API。
L2
Inference Engines: vLLM、SGLang、TensorRT-LLM、LMDeploy、Ollama、llama.cpp 六路热插拔，统一 OpenAI-compatible 外观但不绕过内部 Router。
L1
Compute Frameworks: PyTorch、TensorRT-LLM、JAX、Flax、Triton、TensorRT、ONNXRuntime 等，按硬件和工作负载注册能力。
L0
Infrastructure Baseline: Kubernetes 1.35.4 + Cilium 1.19.3 + containerd 2.2.3 + Rainbond 6.7.1；硬件基线为 2 x NVIDIA DGX Spark，ConnectX-7 200GbE，NVMe 与 S3 兼容对象存储。

4. 请求生命周期

用户上传 DWG/IFC/Office/PDF/业务数据
  -> L7 Next.js / React 模块工作台
  -> L5 axum API Gateway: Auth, Rate Limit, Audit
  -> L4 LangGraph Planner: task decomposition
  -> L3 Rust Harness Core: file runtime, parser, router, schema
  -> L2 InferenceRouter: vLLM/SGLang/TensorRT-LLM/etc.
  -> Planner -> Generator -> Evaluator -> RuleChecker -> SchemaValidator -> Approver
  -> SSE / event stream / artifact version / audit record
  -> L7 CDE workbench shows status, evidence, approval and archive state

9. 生命周期与审批状态机

draft -> submitted -> generating -> evaluating -> rule_checking
  -> schema_validating -> pending_approval -> approved -> archived

异常状态:
  rejected, blocked

事件:
  create, submit, generate, evaluate, rule_check, validate_schema,
  request_approval, approve, reject, archive, reopen, block, resolve_blocker

交付物按钮语义固定为: 生成、评估、校核、Schema、审批、归档。每次 action 必须返回更新后的 artifact、action message 和 audit event。

10. AI 与 Agent 架构

组件 职责
Planner 任务拆解、输入选择、工具选择、工作流路径规划。
Generator 生成设计、模型、BOQ、报告、流程项或归档包。
Evaluator 独立复核生成结果，不能与 Generator 合并。
RuleChecker 运行确定性业务、工程、安全、法律和组织规则。
SchemaValidator 验证 JSON Schema、IFC Schema、Module Schema、OpenAPI、AsyncAPI。
Approver 人工或自动发布门禁，负责状态、签审和归档边界。
ModelRouter / InferenceRouter 模型选择、白名单、成本、SLA、fallback、本地与远程推理抽象。
ToolRouter / WorkflowRouter 工具权限、沙箱、执行、审计、模块事务和 DAG 编排。

支持适配方向包括本地 Ollama、LM Studio、Hugging Face 本地缓存/TGI、vLLM、SGLang、TensorRT-LLM、LMDeploy、llama.cpp，以及 OpenAI-compatible API、Hugging Face Inference Endpoints、OpenRouter 等外部 provider adapter。

13. 后端原生文件运行时

工程格式优先级固定为: source bytes / entity graph / vector / B-Rep / properties -> lightweight native cache manifest -> OpenUSD / USDZ / 3D Tiles -> glTF / GLB audited fallback -> explicit failure。

格式族 主路线 真源与派生边界
IFC / openBIM IfcOpenShell / ThatOpen fragments worker / buildingSMART Validate。 源 IFC 是 record；OpenUSD/USDZ、3D Tiles、GLB、fragments 和 properties index 是缓存或派生。
DWG / DXF PanCAD / MLightCAD @mlightcad/cad-simple-viewer；DWG 经 LibreDWG WASM 隔离边界。 源图纸、图层、实体、颜色、字体、命令边界是真源；PDF、截图、SVG 重绘不能冒充 CAD 支持。
STEP / IGES / BREP OCCT/OCP/FreeCAD-compatible B-Rep worker。 B-Rep、拓扑和属性优先；mesh 只作为显示缓存。
3DM / Rhino / Grasshopper rhino3dm/OpenNURBS worker，必要时 licensed Rhino/Grasshopper sidecar。 保留 NURBS、图层、材质、属性和 source ids。
SKP / SketchUp Licensed SketchUp Ruby sidecar、SketchUp IFC Manager、Yulio glTF exporter、Speckle Connector。 IFC 是 openBIM exchange derivative；GLB 是 source-bound runtime derivative。
PDF / Office Office: Collabora WOPI 隔离服务；PDF: Stirling-PDF sidecar + PaddleOCR/MinerU worker。 源 Office/PDF 是 record；HTML、OCR、Markdown、图片或 PDF 派生不能替代原生编辑和保存回写。
代码 / 配置 / 文本 PanCode: Monaco 0.55.1 inline editor、code-server 4.121.0 sidecar、tree-sitter 0.26.9。 保存回写必须走 /api/local-files/{fileId} 或 /code-session/commit，更新版本、checksum 和审计。
