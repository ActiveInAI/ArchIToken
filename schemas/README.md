# ArchIToken multi-schema source of truth (§4)

The ArchIToken constitution requires schema-driven engineering across API,
events, agents, BIM/IFC, and modules. This directory is the published schema set;
each schema has a single, authoritative producer so the contract never drifts
from the code.

| Schema | File | Source of truth | How it stays in sync |
|---|---|---|---|
| HTTP API | `04-backend/openapi.yaml` | hand-authored gateway contract | `redocly lint` + SDK generation (OpenAPI CI job) |
| Agent I/O (JSON Schema) | `schemas/agent-output.schema.json` | Pydantic models in `architoken_agent.state` | generated by `04-backend/agent-orchestrator/scripts/export_schema.py`; drift-checked in CI |
| Events (AsyncAPI) | `schemas/asyncapi.yaml` | `data_event_outbox` rows from the module-operation runtime trigger | structural validation in `scripts/check-schemas.sh` |
| Module (JSON Schema) | `schemas/module.schema.json` | Module Registry (`module_registry.rs` + `module-registry.ts`) | every registered module validated in `03-frontend/lib/module-registry.test.ts` |
| IFC / openBIM | `schemas/ifc-validation.md` | `06-workers` IFC worker + SJG 157 + IDS | worker round-trip test + `check-schemas.sh` |

## Regenerating the agent schema

```bash
cd 04-backend/agent-orchestrator
uv run python scripts/export_schema.py   # rewrites ../../schemas/agent-output.schema.json
```

`agent-output.schema.json` is **generated**, not hand-edited. CI re-runs the
exporter and fails on any diff (`scripts/check-schemas.sh`), which doubles as the
schema diff / backward-compatibility guard: a contract change in
`architoken_agent.state` must be accompanied by the regenerated schema.

## Frontend SDK / types strategy

- HTTP types: generated/validated from `openapi.yaml` (OpenAPI CI job).
- Agent I/O types: `03-frontend/lib/api.ts` mirrors `agent-output.schema.json`
  (`AgentInvokeResponse` / `AgentGateResult` / `AgentGateFinding`); the JSON
  Schema is the cross-language contract the gateway passes through transparently.
- Module specs: validated against `module.schema.json` in `module-registry.test.ts`.

## Agent outputs are JSON Schema validated

The orchestrator validates every request/response at the Pydantic boundary at
runtime; `agent-output.schema.json` is the externally publishable form of that
same contract, kept identical by the CI drift check.

## CI

`scripts/check-schemas.sh` (CI job *Schema source-of-truth (§4)*):

1. every schema file parses (JSON / YAML);
2. `agent-output.schema.json` matches a fresh export (no drift);
3. `module.schema.json` is a valid JSON Schema document.
