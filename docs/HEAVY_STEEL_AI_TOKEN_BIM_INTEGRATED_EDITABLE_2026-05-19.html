01

执行摘要与总体战略

重钢企业的 AI 与 BIM 升级不应停留在销售问答、图纸识别或单点看板，而应形成“市场获客 → 计划立项 → 方案建模 → 深化冻结 → 计量造价 → 生产制造 → 材料物流 → 施工验收 → 数字孪生 → 数字档案 → 财务人力 → AI 中台 → 设置治理”的全链条业务操作系统。

1内部提效减少重复建模、重复算量、重复录入、资料查找和跨部门等待。
2业务闭环让模型、图纸、清单、工单、物流、施工、档案和财务互相校核。
3数据资产沉淀构件、节点、清单、工艺、质量、客户、合同和案例库。
4AI 中台统一模型路由、知识库、工具插件、审计、额度和安全策略。
5产品封装把内部成熟能力做成客户门户、SaaS、API、报告和项目服务包。
6商业盈利通过订阅、API、AI 服务额度、项目实施、运维和成果分成变现。

1.1 整体目标

一模到底以 BIM / IFC / 构件编码为主线，贯通方案、深化、造价、制造、物流、施工和档案。
一账到底每次 AI 调用、每个项目、每个客户、每个部门都能记录额度消耗、实际成本和收益归属。
一审到底所有关键输出经过 Planner → Generator → Evaluator → RuleChecker → SchemaValidator → Approver，不允许 AI 自评自批。
一权到底客户、项目、岗位、部门、数据密级、地域与服务套餐共同决定可访问范围。
一证到底模型版本、审批、问题、变更、质量、签收、发票、合同和档案形成可追溯证据链。
一商到底内部能力成熟后，封装为可报价、可交付、可续费、可审计的客户产品。

1.2 商业化总公式

AI 商业净利润 = 内部降本收益 + 销售增收收益 + 客户订阅收入 + API 调用收入 + AI 服务额度收入 + 项目增值服务收入 + 成果分成收入 - 外部模型 API 成本 - 本地模型与算力成本 - 数据治理成本 - 研发运维成本 - 客服销售成本 - 合规与税费成本 - 退款与坏账损失

1.3 战略判断

  1. AI 不是独立销售工具，而是业务链条的辅助生产力、审计链和商业化服务能力。
  2. BIM 不是静态成果，而是模型、构件、成本、工艺、状态、质量和档案的业务数据载体。
  3. Token 不应被做成虚拟币或投资品，只能作为平台内 AI 服务额度或调用点数。
  4. 通用模型 API 用于快速启动和非敏感任务，本地模型用于敏感数据、企业知识和长期成本优化。
  5. 所有专业判断必须保留责任人、标准来源、证据链和审批状态。

本体系的工程执行底座是 BIM 主模型。主模型不只是设计成果，而是方案、深化、造价、生产、物流、施工和档案共同使用的全过程业务数据载体。下游业务必须引用有效模型版本、构件编码和审批状态。

04

AI 中台、模型路由与数据架构

推荐采用“外部大模型 API + 中国大陆本地模型 + 行业知识库 + 业务插件 + AI 网关 + 服务额度计量 + 客户门户”的混合架构。外部 API 适合非敏感通用任务，本地模型适合项目图纸、成本、生产、财税、人力等敏感数据。

4.1 总体架构

客户门户 / 企业内部门户 / 移动端 / API 开放平台
  ↓
AI 网关：认证、权限、限流、脱敏、审计、计量、账单
  ↓
WorkflowRouter / ToolRouter / ModelRouter / InferenceRouter
  ├─ 外部大模型 API：客服、翻译、文案、普通报告、公开资料分析
  ├─ 中国大陆本地大模型：图纸、BIM、合同、成本、财务、人力、档案
  ├─ OCR / 视觉模型：图纸识别、构件表提取、现场照片、质检图像
  ├─ 预测模型：成本、工期、质量、现金流、退费、成交概率
  ├─ 规则引擎：权限、计费、退费、审批、风控、数据分级
  └─ RAG 知识库：标准、合同、图纸、节点、工艺、项目档案
  ↓
业务系统：CRM / BIM / ERP / MES / WMS / QMS / 财务 / HR / 档案 / IoT
  ↓
审计、服务额度账单、客户订阅、API 计费、项目利润核算

4.2 模型路由规则

任务类型 推荐路由 理由 审批要求
多语言客服、销售邮件、公开资料摘要外部 API 优先通用能力强，上线快，适合全球客户沟通。客服主管抽检，敏感词与报价边界审核。
BIM 模型、深化图纸、构件清单本地模型优先涉及图纸、结构和项目敏感资料。专业负责人复核，不得直接作为施工依据。
合同底价、报价成本、供应商价格本地模型涉及商业秘密和客户底价。商务 / 财务审批。
生产工艺、质量、工厂排产本地模型 + 规则引擎涉及企业核心工艺和制造能力。生产负责人确认。
财务、税务、退税、补贴本地模型 + 专项规则涉及敏感财税资料和监管责任。财务税务负责人确认。
人力薪酬、绩效、岗位权限本地模型涉及个人信息和劳动合规。HR 与组织授权审批。
结构、施工、安全建议AI 辅助 + 专业复核涉及生命安全、结构安全和施工安全。注册人员、项目负责人或安全负责人审批。

4.3 AI 中台必备能力

模型路由按任务、数据密级、成本、延迟、法域和套餐选择模型。
知识库 RAG检索企业标准、历史项目、节点库、工艺库、合同库、档案库后再回答。
业务插件调用 BIM、ERP、MES、WMS、QMS、CRM、财务、人力和档案系统 API。
AI 门禁链Planner、Generator、Evaluator、RuleChecker、SchemaValidator、Approver 分离。
服务额度计量记录客户、项目、部门、用户、任务、模型、输入输出量和费用。
审计留痕记录来源、模型版本、提示词版本、调用人、输出状态、审批状态和写回动作。
数据分级按公开、内部、项目敏感、高度敏感、个人信息、跨境数据设策略。
内容安全输入输出审核、风险词、专业责任声明、投诉处置与日志留存。
成本控制限额、缓存、低成本模型、熔断、失败不计费和异常告警。

4.4 AI 输出状态

状态 含义 是否可作为正式依据

rule_checked已通过机器规则和基础 Schema 检查。否，仍需责任人确认。

05

AI 服务额度计量、账单与盈利机制

本体系中的“Token”只作为 AI 服务计量单位，不是虚拟币、代币、证券、积分投资品或可交易资产。建议对外统一称为“AI 服务额度”或“AI 调用点数”，后台可以把不同模型、OCR、图纸解析、RAG 检索、业务系统 API 调用折算为统一计量口径。

5.1 生产与折算逻辑

外部大模型 API 额度
  + 本地 GPU 推理能力
  + OCR / 视觉 / 预测模型能力
  + BIM / 图纸 / 文档解析能力
  + 向量知识库检索能力
  + 业务系统 API 调用能力
  + 存储、审计、导出和报告生成能力
      ↓
AI 网关统一计量
      ↓
折算为企业内部额度 / 项目额度 / 客户额度 / API 调用点数
      ↓
内部成本分摊 + 对外套餐销售 + 项目利润核算
      ↓
形成账单、发票、退款、利润分析和客户续费策略

5.2 账户体系

账户类型 对象 计量口径 管理目标
企业总账户公司 AI 中台。总 API 额度、本地算力、模型调用、总额度池。控制总体 AI 成本。
部门账户市场、方案、深化、造价、生产、物流、施工、财务、人力。部门月度额度、任务额度、超额申请。内部限额与成本分摊。
项目账户具体工程项目或加工订单。项目 AI 使用量、模型成本、收益归属。核算项目真实利润。
客户账户业主、总包、设计院、分包、供应商、会员客户。购买额度、套餐额度、超额计费、退款。形成对外收入。
API 账户外部系统、合作伙伴或客户系统。调用次数、并发、数据量、失败率、SLA。形成 API 商业化收入。

5.3 计费对象与成本字段

计费对象 可计量字段 注意事项
文本模型输入量、输出量、模型等级、上下文长度、工具调用次数。区分外部 API 成本与本地算力成本。
图纸 / OCR页数、图片尺寸、图层数量、识别字段数量、人工复核次数。失败识别和人工复核应单独计费或补偿。
BIM / 模型解析文件大小、构件数量、属性数量、转换耗时、校验报告数量。绑定源文件校验和、模型版本和构件编码。
知识库检索检索次数、向量库大小、命中文档、重排次数。记录来源，不把检索结果当专业结论。
业务系统 API接口调用次数、数据行数、写回次数、审批次数。写回正式系统前必须有责任人确认。
报告和导出报告页数、附件数量、导出格式、签章和归档状态。正式报告需保留版本和审批记录。

5.4 红线与推荐命名

不建议叫法 建议叫法 原因
Token 币AI 服务额度避免被理解为虚拟货币或代币。
算力币AI 算力点数强调其是服务计量单位。
平台币AI 调用点数避免交易、兑换、炒作风险。
可交易 Token不可转让服务额度明确只能在平台内消费。
充值 Token购买 AI 服务包更符合软件服务、数据处理和技术服务表述。
不得提现客户购买额度不得兑换现金。
不得交易不得建立二级市场或用户间转让机制。
不得承诺升值不得宣传投资收益、涨价收益或理财属性。
不得脱离服务额度必须对应真实 AI、API、数据处理或软件服务。

客户产品体系与收费模式

对外产品不应直接卖“模型能力”，而应卖客户可理解、可验收、可续费的工程服务。每个产品都必须绑定功能边界、服务额度、人工复核、责任声明、交付物和退款规则。

07

合规、安全、权限与责任边界

AI 商业化必须同时处理专业责任、数据安全、个人信息、跨境传输、内容安全、合同责任、财税规则、开票与退款。未绑定明确法规、标准、条文和证据时，只能输出经验建议，不能输出“合规”“可施工”“可报审”“可验收”“可入账”等结论。

7.1 中国大陆生成式 AI 与平台经营关注点

问题 判断重点 建议控制
是否面向公众是否开放注册，是否任何用户都可使用。先做企业客户封闭服务，明确客户组织与授权。
是否生成内容是否生成文本、报告、报价、建议、图纸说明。加内容安全审核、免责声明和输出状态。
是否有舆论属性是否可能影响公众意见或传播内容。关注算法备案和生成式 AI 服务要求。
是否涉及专业判断结构、税务、法律、施工、安全、造价等。必须人工复核，不得宣称 AI 替代专家。
是否经营网站 / SaaS / API是否有客户门户、在线交易、公开 API、用户账号。结合实际确认 ICP、等保、日志、协议、隐私政策和数据处理协议。

7.2 数据分级与模型使用

数据等级 数据内容 处理方式 外部 API 适用性
公开数据公开市场价格、公开政策、公开案例、宣传资料。可用于外部模型与客户服务。适合。
内部数据标准节点、工艺标准、内部流程、非敏感项目数据。仅内部员工访问，必要时脱敏。有条件。
项目敏感数据图纸、BIM 模型、合同、报价、成本、进度。按项目权限访问，本地模型或私有云优先。谨慎。
高度敏感数据财务、薪酬、利润、客户底价、税务、商业秘密。本地模型优先，严格审批和审计。不建议。
个人信息客户联系人、员工信息、手机号、邮箱、支付信息。最小必要、授权、脱敏、访问日志和出境评估。谨慎。

7.3 专业责任边界

AI 输出 允许用途 不得替代
报价建议辅助销售测算、成本分析和客户沟通。不得替代正式报价审批、合同定价和财务审核。
设计建议辅助方案比选、深化校核和问题提示。不得替代注册建筑师、注册结构工程师或专业负责人签认。
造价测算辅助清单匹配、成本预测和变更分析。不得替代正式结算、审计确认和注册造价工程师职责。
生产排产建议辅助产能分析、工序安排和质量预警。不得替代生产负责人、质量负责人和安全制度。
施工建议辅助交底、整改和现场问题归类。不得替代项目负责人、安全负责人、监理和验收责任。
税务退税建议辅助资料检查、政策提示和风险预警。不得替代税务申报责任人、税务师或主管机关意见。
人力绩效分析辅助排班、培训和管理决策。不得作为唯一考核、奖惩、劳动争议处理依据。

7.4 全球数据区域与部署建议

客户区域 部署与数据建议 重点风险

欧盟欧盟区域数据中心或合规云优先。GDPR、数据主体权利、跨境传输。
澳洲澳洲或客户合同允许的合规区域云服务。隐私保护、本地客户合同要求。
美国美国区域模型服务和数据中心，合同明确责任边界。州隐私法、商业合同责任、出口管制。
中东关注当地数据本地化和政府项目要求。数据主权、政府项目安全要求。
东南亚按国家区分数据合规、付款方式和税务要求。个人数据保护、跨境传输、支付合规。

合同、财税、发票、退款与多币种

客户侧 AI 产品必须有清晰合同表述、收入确认规则、用量明细、失败补偿、退款条件、汇率规则和税务审核。具体税目、开票口径和收入确认必须由财务、税务顾问和所在地主管税务机关确认。

8.1 推荐合同与发票表述

软件服务客户门户、SaaS 订阅、账号权限、模块使用。
信息技术服务系统集成、API 接入、私有化部署、运维支持。
数据处理服务图纸识别、资料 OCR、模型解析、档案整理。
AI 辅助分析服务报价辅助、模型审查辅助、工程量复核辅助。
平台服务客户协同、构件追踪、项目看板、数字孪生服务。
技术咨询服务方案比选、流程梳理、数据治理、客户培训。

8.2 收入确认建议

产品类型 收入确认建议 说明
AI 服务额度包按实际消耗、服务期间或有效期摊销。避免一次性确认后产生退款争议。
SaaS 订阅按服务期间确认。月费或年费按期间分摊。
API 调用按实际调用量确认。调用日志必须可审计。
项目服务按项目节点确认。如方案、上线、验收、运维节点。
私有化部署按交付和验收节点确认。合同应明确验收标准。
成果分成按实际成功金额确认。如退税、补贴、索赔、节约成本。

8.3 退款规则

事项 建议规则 原因
未使用额度可按合同约定退还、延期或转为下期额度。提高客户接受度。
已消耗额度原则上不退，除非平台故障或合同另有约定。AI 成本已实际发生。
API 失败失败调用不计费或补偿额度。降低争议。
客户资料错误已发生分析成本仍可收费，但需在协议中明确。控制服务责任。
AI 结果不作为最终依据合同中明确 AI 仅为辅助建议。控制专业责任。
退款汇率提前约定按原付款币种、原汇率或指定退款日汇率。控制跨境退款争议。

8.4 多币种结算字段

字段 说明 用途
客户币种RMB、USD、EUR、AUD、HKD、SGD、AED 等。报价和收款。

付款汇率客户实际付款时汇率。实际收入确认。

接口字段、数据闭环与审计证据

AI 与业务系统融合必须形成“读取数据 → 生成建议 → 人工确认 → 写回系统 → 记录额度 → 归档证据”的闭环。AI 输出不得直接覆盖正式业务数据。

系统 接入数据 AI 用途

造价系统清单、计量规则、单价、成本科目、变更。报价、成本测算、变更测算、结算复核。
ERP采购、库存、供应商、合同、发票、付款。采购预测、材料成本、库存预警、财务联动。
MES排产、工序、工时、设备、加工状态。排产优化、工时预测、产能预警。
WMS仓储、包装、装车、发运、到场、堆场。物流优化、构件追踪、到场齐套。
QMS质检、返修、NDT、涂装、防火、验收。质量风险预测、返工归因、验收资料生成。
施工管理交底、进度、问题、整改、验收、照片。现场问题 AI、质量识别、竣工资料自动生成。
数字孪生BIM、IoT、状态、进度、质量、物流、运维数据。可视化问答、状态推演、风险预测。
数字档案合同、图纸、变更、签证、验收、发票、竣工资料。自动归档、资料检索、档案问答。
财务 / HR收入、成本、发票、收款、付款、退税、人员、工时、薪酬。利润预测、现金流预测、税务检查、绩效辅助。

9.2 AI 写回规则

  1. AI 输出不得直接覆盖正式业务数据，必须先进入待确认状态。
  2. 责任人确认后，结果才能写回 CRM、BIM、ERP、MES、WMS、QMS、财务或档案系统。
  3. 所有 AI 调用必须保留输入摘要、输出摘要、模型版本、提示词版本、额度消耗、调用人、责任人、时间戳和审批状态。
  4. 涉及报价、结构、造价、税务、施工、安全、人力考核的输出必须设置人工审批。
  5. 模型、图纸、清单、工单、物流、施工和档案的版本关系必须可追溯。

9.3 最小审计字段

字段 说明 用途
audit_id审计事件唯一编号。全链路追踪。
tenant_id / project_id租户、客户、项目或订单。权限隔离和成本核算。
module_id业务模块，如 detailed_design、production_manufacturing。模块审计与报表。
source_file_id / checksum源文件编号和校验和。防止模型、图纸、资料错版。
object_id / component_code构件、清单、工单或问题对象。对象级追溯。
model_id / version调用模型、版本、路由策略。AI 结果可复盘。
input_classification数据密级、是否个人信息、是否跨境。合规与安全审计。
ai_output_statusdraft_assist、rule_checked、professional_review_required 等。防止草稿误用。
quota_cost本次调用消耗的服务额度、实际成本和客户计费。账单与利润核算。
approval_record审批人、意见、时间、结论。责任边界和归档。
