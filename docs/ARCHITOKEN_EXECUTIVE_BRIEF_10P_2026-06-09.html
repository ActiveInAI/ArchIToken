05 / buildingSMART OpenBIM

OpenBIM 标准栈可落地，认证声明必须受控

CDE 是项目共享信息环境与管理过程，不是普通网盘；IFC、IDM、IDS、BCF、bSDD、Validate、OpenCDE/API 与 SJG157 本地化语义字典围绕同一项目信息容器协同。实现标准能力不等于已获 buildingSMART 官方认证。

职责拆分：CDE 管文档、模型、元数据、版本、状态、权限、审批和审计；IFC 表达开放模型；IDM 定义信息交付流程；IDS 声明机器可读交付要求；BCF 管模型协同主题、视点、评论、责任与关闭；bSDD 提供开放分类与属性语义；Validate 输出模型与规则校验；OpenCDE/API 定义跨系统协作接口；SJG 157-2024 把语义字典本地化为建筑、空间、构件、系统类目和编码。

07 / 构件物料 BOM 管线与门禁

导入 → 解析 → 校验 → 计重 → 复核 → 发布，全程可回放

BOM 由固定 Agent 管线驱动，每步只写草稿与证据，发布态由人工审批产生。QTO、MTO、BOM、BOQ、PBOM、MBOM 是不同业务视图，不能用一个 Excel 表替代；下游只读 issued/approved 数据。

Agent 管线：ComponentBomImport（读 XLSX 建 documents/versions/lines）→ ComponentNameParser（6 token）→ CategoryCodeLink（编码自动链接 + 不符告警）→ ComponentBomValidation（阻断/告警规则）→ WeightCalculation（截面解析 + 密度查表，仅建议不直接写）→ ComponentBomReview（汇总阻断项与建议）→ 人工复核审批 → ComponentBomExport（写 issued，导出 xlsx/pdf/审计归档）。

6告警规则：长度/截面/材料/重量/图号/类型歧义
统一公式unit = area·length/1e9·density，按截面解析
